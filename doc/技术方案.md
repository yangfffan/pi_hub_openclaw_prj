# 树莓派远程语音代理 - 技术方案

## 1. 系统概述

本系统旨在实现树莓派与 OpenClaw 的语音交互。树莓派作为语音代理，通过麦克风采集用户语音，经过 ASR 服务转换为文字，发送给 OpenClaw 进行处理；OpenClaw 返回的结果经过 TTS 服务转换为语音，再由树莓派播放。

**注意**：根据[需求文档](./需求文档.md)的更新，ASR/TTS 服务部署在树莓派端，HUB 与树莓派之间只传输文本消息。

## 3. 组件设计

### 3.1 树莓派客户端 (Pi Client)

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 语音唤醒 | (待实现) | 暂时使用按回车模拟唤醒 |
| 录音 | pyaudio | 16kHz采样率，16位，单声道WAV格式 |
| ASR | 腾讯云 ASR | 通过 HTTP API 调用云服务，将语音转为文字 |
| TTS | 腾讯云 TTS | 通过 HTTP API 调用云服务，将文字转为语音 |
| 播放 | pyaudio | 播放接收到的音频 |
| HTTP客户端 | requests | 轮询HUB接口，上传/下载文本消息 |

### 3.2 服务器端 HUB

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| Web框架 | FastAPI | 提供REST API接口 |
| 消息队列 | Redis | 上下行消息队列 |
| OpenClaw集成 | subprocess | 调用CLI命令 |

## 4. 通信协议

### 4.1 树莓派 → HUB

采用 HTTP REST API 轮询机制，轮询间隔 **2秒**。传输内容为**JSON文本**。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/upload` | POST | 上传文字消息 |
| `/api/v1/poll` | GET | 轮询获取下发文字消息 |

### 4.2 HUB → OpenClaw

HUB 通过 subprocess 调用 OpenClaw CLI 命令，轮询间隔 **1秒**。

```bash
# 发送消息给 Agent
openclaw agent --agent <agent_id> -m "<message>" --json

# 读取最近消息
openclaw message read --channel <channel> --limit 10 --json
```

### 4.3 Token 绑定机制

树莓派配置 Token（手工配置），每次请求携带在 HTTP Header 中：

```
Authorization: Bearer <token>
```

HUB 验证 Token 一致后才响应。

## 5. API 接口详细设计

### 5.1 树莓派端接口

#### 5.1.1 上传文字消息

树莓派将 ASR 转换后的文字上传到 HUB。

```
POST /api/v1/upload
Content-Type: application/json
Authorization: Bearer <token>

Body:
{
  "text": "用户说的话",
  "device_id": "pi_living_room"  // 可选
}
```

响应:
```json
{
  "success": true,
  "message_id": "msg_123456"
}
```

#### 5.1.2 轮询下发消息

树莓派轮询获取 HUB 下发的文字消息。

```
GET /api/v1/poll?device_id=<device_id>
Authorization: Bearer <token>
```

响应:
```json
{
  "messages": [
    {
      "id": "msg_123456",
      "text": "LLM的回复",
      "timestamp": 1710123456
    }
  ]
}
```

### 5.2 内部接口 (可选)

#### 5.2.1 设备注册

```
POST /api/v1/device/register
Content-Type: application/json

Body:
{
  "token": "device_token",
  "device_name": "pi_living_room"
}
```

## 6. 数据流设计

### 6.1 树莓派主动流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  唤醒检测 │────▶│  录音    │────▶│  ASR    │────▶│  上传    │
└──────────┘     └──────────┘     │(腾讯云)  │     │  文本    │
                                   └──────────┘     └──────────┘
                                                        │
                                                        ▼
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  播放    │◀────│  TTS     │◀────│  轮询    │◀────│  等待    │
│(pyaudio) │     │(腾讯云)  │     │  回复    │     │  响应    │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
```

详细步骤：

1. **唤醒检测**: (待实现) 暂时使用按回车模拟唤醒
2. **开始录音**: pyaudio 开始录音，检测静音后停止
3. **ASR识别**: 调用腾讯云 ASR 将音频转为文字
4. **上传文本**: POST 文字到 `/api/v1/upload`
5. **轮询回复**: 每2秒调用 `/api/v1/poll` 检查是否有回复
6. **TTS转换**: 调用腾讯云 TTS 将文字转为音频
7. **播放音频**: 通过 pyaudio 播放

### 6.2 服务器主动流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│ OpenClaw │────▶│  消息    │────▶│  存入    │
│ 心跳/定时│     │  检测    │     │  队列    │
└──────────┘     └──────────┘     └──────────┘
                                           │
                                           ▼
┌──────────┐     ┌──────────┐     ┌──────────┐
│  播放    │◀────│  TTS     │◀────│  轮询    │
│(pyaudio) │     │(腾讯云)  │     │  获取    │
└──────────┘     └──────────┘     └──────────┘
```

详细步骤：

1. **消息检测**: HUB 每1秒调用 `openclaw message read` 轮询
2. **判断新消息**: 检测到新消息后，获取消息内容
3. **存入队列**: 将文本消息存入 Redis 下发队列
4. **树莓派轮询**: 树莓派每2秒轮询 `/api/v1/poll` 获取
5. **TTS转换**: 调用腾讯云 TTS 将文字转为音频
6. **播放**: 通过 pyaudio 播放

## 12. 错误处理

### 12.1 网络错误

| 错误类型 | 处理策略 |
|----------|----------|
| 连接失败 | 指数退避重试 (1s, 2s, 4s, 8s, 最大30s) |
| 超时 | 超时时间10秒，超时后跳过本次请求 |
| SSL错误 | 记录日志，允许配置跳过验证（仅开发环境） |

### 12.2 业务错误

| 错误类型 | 处理策略 |
|----------|----------|
| Token 无效 | 返回 401，客户端提示配置错误 |
| 消息格式错误 | 返回 400，提示支持的格式 |
| OpenClaw 不可用 | 记录日志，重试队列消息 |

### 12.3 日志

| 级别 | 输出位置 | 说明 |
|------|----------|------|
| DEBUG | 控制台 | 调试信息 |
| INFO | 控制台 + 文件 | 正常流程 |
| WARNING | 控制台 + 文件 | 警告信息 |
| ERROR | 控制台 + 文件 | 错误信息 |

## 13. 安全考虑

1. **HTTPS/TLS**: 生产环境必须使用 HTTPS
2. **Token 验证**: 所有接口验证 Token
3. **输入验证**: 验证文本消息大小、格式
4. **限流**: 防止恶意请求
5. **凭证安全**: 腾讯云凭证存储在配置文件中，注意文件权限
