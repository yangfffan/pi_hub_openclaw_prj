# 需求文档

树莓派远程语音代理

# 项目概述

- 树莓派作为语音代理，通过麦克风和扬声器实现语音输入输出
- 树莓派可以上网，但处于内网环境，没有公网IP
- 服务器上已安装openclaw。需要写一个HUB服务，用于实现树莓派与openclaw的通信
- 树莓派通过轮询实现与服务器HUB通信，服务器HUB通过localhost连接到服务器上已安装的openclaw gateway，实现消息的转发

# 硬件描述

## 树莓派

- 树莓派5，系统：Raspberry Pi OS
- usb麦克风
- usb音箱

## 服务器

- 系统：Ubuntu 24.04
- 已安装openclaw

# 使用流程

## 树莓派主动

- 树莓派听到语音唤醒词如"小克小克"，则开始录音，检测到录音结束，则停止录音
- 树莓派通过网络ASR服务（腾讯云）将录音文件转换为文字
- 树莓派把文字消息发送给服务器HUB的上传消息队列
- 服务器HUB从上传消息队列读取消息，发给openclaw gateway
- openclaw gateway收到文字，把树莓派当成同一个channel的消息（类似telegram这样的channel），发送给LLM
- LLM收到消息，返回结果，发送给openclaw(openclaw自动实现)
- openclaw收到结果，把文字发给HUB
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（腾讯云）将文字转为语音并播放

## 服务器主动

- openclaw心跳或者定时任务，调用LLM，生成文字发给HUB(openclaw自动实现)
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（如阿里云/腾讯云）将文字转为语音并播放

## 树莓派语音唤醒流程

- 树莓派听到语音唤醒词"小克小克"，唤醒，后播放本地语音"在呢"（zaine.wav）
- 随后树莓派开始录音，检测到录音结束，则停止录音，后播放本地语音"听到啦"（heard.wav）
- 随后树莓派通过网络ASR服务（腾讯云）将录音文件转换为文字，成功后播放本地语音"懂啦"（understood.wav）
- 随后树莓派把文字消息发送给服务器HUB的上传消息队列，成功后播放本地语音"我想一下"（thinking.wav）
- HUB与openclaw处理消息
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息，如果读取到消息，播放本地语音"想好啦"（ok.wav）
- 随后树莓派通过网络TTS服务（腾讯云）将文字转为语音并播放

# 需求更新

1. ASR与TTS在树莓派上通过调用云服务的API实现，即"树莓派-HUB-openclaw"的通信只通过文本

2. 设计测试接口或测试方案，可以单独测试以下过程

- 树莓派<->ASR与TTS云服务（语音->文字，文字->语音）
- 树莓派<->HUB消息通信（树莓派->HUB，HUB->树莓派）
- HUB<->openclaw消息通信（HUB->openclaw，openclaw->HUB）

3. 对于ASR与TTS云服务的测试，可以生成文字，使用一次"文字->语音->文字"的转换，对比转换得到的文字是否一致

4. 代码分为hub（服务器）与pi（树莓派）两个文件夹。在两个文件夹下再分别写一个.md文件说明代码结构与部署与测试流程

5. 测试架构：在hub/test/下放hub与openclaw的测试，在pi/test/下放树莓派和TTS+ASR云服务的测试，在test文件夹放系统测试

- 由于树莓派与服务器环境相似，直接在服务器上进行所有测试

6. 在pi文件夹新增gen_assets.py文件，用于根据yaml文件夹的音色等配置，生成所需的wav文件，放到pi/assets文件夹

- 生成文件包括：
| 文本内容 | 文件名 |
|---------|----------|
| 树莓派 ↔ TTS云服务 | 文字→语音 |
- "在呢"（zaine.wav）
- "听到啦"（heard.wav）
- "懂啦"（understood.wav）
- "我想一下"（thinking.wav）
- "想好啦"（ok.wav）

7. hub与pi的配置分别放在自己的yaml文件中，比如配对token和通信ip等

8. 本地开发使用uv，部署python环境也使用uv安装，把使用流程也更新到readme里
