# 需求文档

树莓派远程语音代理

# 项目概述

- 树莓派作为语音代理，通过麦克风和扬声器实现语音输入输出
- 树莓派可以上网，但处于内网环境，没有公网IP
- 服务器上已安装openclaw。需要写一个HUB服务，用于实现树莓派与openclaw的通信
- 树莓派通过轮询或长连接实现与服务器HUB通信，服务器HUB通过localhost连接到服务器上已安装的openclaw gateway，实现消息的转发

# 硬件描述

## 树莓派

- 树莓派5，系统：Raspberry Pi OS
- usb麦克风
- usb音箱

## 服务器

- 系统：Ubuntu 24.04
- 已安装openclaw

# 使用流程

## 树莓派主动

- 树莓派听到语音唤醒词如"小克小克"，则开始录音，检测到录音结束，则停止录音
- 树莓派通过网络TTS服务（如阿里云/腾讯云）将录音文件转换为文字
- 树莓派把文字消息发送给服务器HUB的上传消息队列
- 服务器HUB从上传消息队列读取消息，发给openclaw gateway
- openclaw gateway收到文字，把树莓派当成同一个channel的消息（类似telegram这样的channel），发送给LLM
- LLM收到消息，返回结果，发送给openclaw(openclaw自动实现)
- openclaw收到结果，把文字发给HUB
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（如阿里云/腾讯云）将文字转为语音并播放

## 服务器主动

- openclaw心跳或者定时任务，调用LLM，生成文字发给HUB(openclaw自动实现)
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（如阿里云/腾讯云）将文字转为语音并播放

## 树莓派语音唤醒流程

- 树莓派听到语音唤醒词"小克小克"，唤醒，后播放本地语音"在呢"（zaine.wav）
- 随后树莓派开始录音，检测到录音结束，则停止录音，后播放本地语音"听到啦"（heard.wav）
- 随后树莓派通过网络TTS服务（如阿里云/腾讯云）将录音文件转换为文字，成功后播放本地语音"懂啦"（understood.wav）
- 随后树莓派把文字消息发送给服务器HUB的上传消息队列，成功后播放本地语音"我想一下"（thinking.wav）
- HUB与openclaw处理消息
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息，如果读取到消息，播放本地语音"想好啦"（ok.wav）
- 随后树莓派通过网络TTS服务（如阿里云/腾讯云）将文字转为语音并播放

# 问题&回答

## Round 1

1. 通信协议细节

- 长连接 vs 轮询：文档提到"轮询或长连接"，需要确定用哪种方案？
- 如果用WebSocket长连接，需要确认openclaw gateway是否支持
- 回答：openclaw gateway只与HUB通信，这个可以使用openclaw gateway HTTP API
- 如果用HTTP轮询，频率（文档说2s）是否满足实时性要求？
- 回答：树莓派在内网，所以树莓派轮询HUB，满足实时性要求

2. 消息队列实现

- 上传/下发消息队列：文档提到两个队列，但未明确：
- 使用什么技术？Redis？RabbitMQ？文件？数据库？
- 回答：可以使用Redis
- 消息的持久化策略？
- 回答：消息在openclaw中作为channel，openclaw自己会保存聊天记录，不需要在HUB持久化

3. ASR/TTS服务

- 具体使用哪家："如阿里云/腾讯云"是示例，需要确定：
- 实际使用哪个厂商的TTS/ASR服务？
- 是否有账号/密钥？还是用开源方案（如Whisper + Coqui TTS）？
- 回答：这个现在可以知道通过HTTP接口调用就行，现在可以先跳过留空

4. openclaw集成细节

- 连接方式：文档说"通过localhost连接到openclaw gateway"
- openclaw gateway的API接口格式是什么？
- 消息体结构（JSON格式）？
- 是否需要认证？
- 回答：openclaw gateway HTTP API 在网上可以查到，你设计之前先去搜索一下。可以参考~/work/edict项目与openclaw的通信方式

5. 树莓派端

- 语音唤醒方案：用哪家？Snowboy？Porcupine？还是其他？
- 回答：你先选一个简单的
- 音频格式：录音用什么编码（PCM/WAV/MP3）？采样率/比特率？
- 播放机制：收到语音文件后如何播放？用哪个库？
- 回答：参考doc/pi_audio_record.py与doc/pi_audio_play.py

6. 网络架构

- 树莓派如何访问服务器？VPN？内网穿透？SSH隧道？
- TLS加密？是否需要？
- 回答：树莓派通过公网访问服务器，服务器有公网IP，可以使用TLS或https

7. 错误处理与日志

- 断网重连机制？
- 日志级别和输出位置？
- 回答：都可以，你看着办

8. 扩展性

- 是否支持多设备（多个树莓派）？
- 是否需要设备注册/绑定机制？
- 回答：树莓派和HUB设计一个token绑定机制，token一致才响应

这个测试是指：
