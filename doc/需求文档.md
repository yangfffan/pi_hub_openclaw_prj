# 需求文档

树莓派远程语音代理

# 项目概述

- 树莓派作为语音代理，通过麦克风和扬声器实现语音输入输出
- 树莓派可以上网，但处于内网环境，没有公网IP
- 服务器上已安装openclaw。需要写一个HUB服务，用于实现树莓派与openclaw的通信
- 树莓派通过轮询实现与服务器HUB通信，服务器HUB通过localhost连接到服务器上已安装的openclaw gateway，实现消息的转发

# 硬件描述

## 树莓派

- 树莓派5，系统：Raspberry Pi OS
- usb麦克风
- usb音箱

## 服务器

- 系统：Ubuntu 24.04
- 已安装openclaw

# 使用流程

## 树莓派主动

- 树莓派听到语音唤醒词如"小克"，则开始录音，检测到录音结束，则停止录音
- 树莓派通过网络ASR服务（腾讯云）将录音文件转换为文字
- 树莓派把文字消息发送给服务器HUB的上传消息队列
- 服务器HUB从上传消息队列读取消息，发给openclaw gateway
- openclaw gateway收到文字，把树莓派当成同一个channel的消息（类似telegram这样的channel），发送给LLM
- LLM收到消息，返回结果，发送给openclaw(openclaw自动实现)
- openclaw收到结果，把文字发给HUB
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（腾讯云）将文字转为语音并播放

## 服务器主动

- openclaw心跳或者定时任务，调用LLM，生成文字发给HUB(openclaw自动实现)
- HUB收到文字消息，存入下发消息队列
- 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息
- 树莓派通过网络TTS服务（如阿里云/腾讯云）将文字转为语音并播放

## 树莓派语音唤醒流程

- 日常循环：树莓派程序一直运行，并进行音量检测。当音量大于阈值，开始唤醒词检测。如果五秒内没有出现唤醒词，回到音量检测状态。
- 唤醒词检测：检测到语音唤醒词"小克"就唤醒。唤醒后播放本地语音"在呢"（zaine.wav），然后开始录音。
- 录音：录音的同时进行人声检测，当人声停止5s后，结束录音。如果人声持续时间超过30s，也结束录音。结束录音后都播放本地语音"听到啦"（heard.wav）
- 录音成功后：
1. 树莓派通过网络ASR服务（腾讯云）将录音文件转换为文字，成功后播放本地语音"听懂啦"（understood.wav）
2. 树莓派在返回的文字后增加内容“（本消息通过语音设备发送，请回复也适合语音播放，仅用自然语言文字消息，内容简短，不要包含列表或链接等难以被语音播放理解的内容）”
3. 随后树莓派把文字消息发送给服务器HUB的上传消息队列，成功后播放本地语音"我想一下"（thinking.wav）
4. HUB与openclaw处理消息
5. 树莓派通过轮询（如2s一次）从HUB消息队列中读取文字消息，如果读取到消息，播放本地语音"想到啦"（ok.wav）
6. 随后树莓派通过网络TTS服务（腾讯云）将文字转为语音并播放
- 完成后回到日常循环
